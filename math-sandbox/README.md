# 🧮 Math Sandbox

A highly streamlined, secure Python execution environment designed as a tool for AI bots. It enables the safe execution of arbitrary Python code with a focus on mathematical and logical operations, pre-packaged with essential libraries (`NumPy`, `SymPy`, `SciPy`).

## 🛡️ Security Features (Five-Level Protection)

1. **Docker Container Limits**: Restricts total processes (`pids_limit`), memory, CPU usage, drops all capabilities (`cap_drop`), and mounts a read-only root filesystem.
2. **Non-Root Execution**: Runs the service under a dedicated, unprivileged non-root user (`sandboxuser`).
3. **Linux Resource Limits**: Employs the `resource` module within Python to restrict virtual memory (`RLIMIT_AS`), CPU time (`RLIMIT_CPU`), and child processes (`RLIMIT_NPROC`) for each execution to prevent memory exhaustion and fork bombs.
4. **Python Subprocess Isolation**: Executes arbitrary user code in a distinct restricted subprocess, preventing it from interfering with the main API service.
5. **Network Isolation**: By default, the container operates on an isolated internal network (`internal: true`), completely blocking internet access. *This is configurable in `docker-compose.yml`.*

## 🚀 Quick Start

### Prerequisites
- Docker
- Docker Compose

### Deployment

Clone the repository (or create the directory structure) and run:

```bash
cd math-sandbox
docker-compose up -d --build
```

The API will be available at `http://localhost:8000`.

## 📖 API Documentation

### Execute Code

Executes the provided Python code in the sandbox.

- **Endpoint**: `/execute`
- **Method**: `POST`
- **Content-Type**: `application/json`

#### Request Body

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `code` | `string` | **Required** | The Python code to execute. |
| `timeout` | `integer` | `5` | Maximum CPU time allowed for execution (in seconds). |
| `memory_limit_mb` | `integer`| `256` | Maximum virtual memory allowed (in Megabytes). |

**Example Request:**

```json
{
  "code": "import numpy as np\nprint(np.array([1, 2, 3]) * 2)",
  "timeout": 3,
  "memory_limit_mb": 128
}
```

#### Response

| Field | Type | Description |
| :--- | :--- | :--- |
| `stdout` | `string` | Standard output generated by the code execution. |
| `stderr` | `string` | Standard error or exceptions generated by the code execution. |
| `exit_code` | `integer`| Process exit code (0 for success, non-zero for errors/timeouts). `137` commonly indicates a timeout/SIGKILL. |

**Example Response (Success):**

```json
{
  "stdout": "[2 4 6]\n",
  "stderr": "",
  "exit_code": 0
}
```

**Example Response (Timeout):**

```json
{
  "stdout": "",
  "stderr": "Warning: Failed to set resource limits...\nExecution timed out (Wall-clock limit reached).",
  "exit_code": 137
}
```

### Health Check

Checks if the API is running.

- **Endpoint**: `/health`
- **Method**: `GET`

#### Response

```json
{
  "status": "ok"
}
```

## ⚙️ Configuration

### Enable Internet Access
By default, the sandbox cannot access the internet to prevent data exfiltration or malicious downloads. To allow network access, edit the `docker-compose.yml` file:

```yaml
networks:
  sandbox_net:
    internal: false # Change this from true to false
```

Then, restart the service:
```bash
docker-compose down
docker-compose up -d
```
